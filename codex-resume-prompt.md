# 给 Codex 的续开发指令

请继续开发 SRMS（Smart Rental Management System）。

当前状态：Task001 项目初始化已经完成，前后端构建、后端单元测试、接口测试、ESLint 和 Prisma 校验均已通过。现在从 Task002“登录功能”开始。

开始前必须依次读取：

1. `docs/requirements-freeze-v1.md`
2. `docs/database-design.md`
3. `docs/system-permissions-audit-design.md`
4. `README.md`
5. 当前前后端源代码和 `package.json`

执行规则：

- `SRMS-RB-1.0` 是冻结需求；不要重新询问已经确定的规则。
- 文字冻结需求优先于 HTML 原型。原型漏画功能不代表需求被取消。
- 每次只完成一个 Task，但该 Task 内需要同时完成数据库、后端、前端和测试。
- 不得把多个业务 Task 混在一次修改中。
- 所有数据库变更通过 Prisma Migration。
- 后端负责真正的权限校验，不能只隐藏前端按钮。
- 统一接口格式为 `{ code, message, data }`。
- 金额使用精确小数，禁止浮点金额计算。
- 不读取、提交或展示 `.env`、真实租客资料、交易截图和数据库备份。
- 修改前检查现有文件和未提交变更，不覆盖用户已有工作。
- 完成后执行生产构建、单元测试、接口测试、ESLint 和 Prisma 校验。
- 更新 README 的任务进度，并建立 Git 提交；若环境不允许提交，生成不含依赖、构建产物和密钥的源码快照。

Task002 验收范围：

- 初始化第一名超级管理员，系统必须始终保留至少一名启用状态的超级管理员。
- 登录名和密码登录。
- 密码采用安全哈希，不保存明文。
- JWT 短期访问令牌与刷新令牌；数据库只保存刷新令牌哈希。
- 刷新令牌轮换、退出登录和全部设备退出。
- 连续失败次数、临时锁定和最后登录时间。
- 登录页、会话状态管理、路由保护和退出入口。
- 不在登录接口返回提成、财务或其他无关业务字段。
- 登录、失败、锁定、刷新和退出的自动化测试。

请先简要报告检查结果，然后直接实施 Task002；只有遇到会改变冻结业务规则的选择时才向我提问。
