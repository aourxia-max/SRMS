# SRMS 系统设置、权限与审计设计（V1.0）

> 文档状态：第一版冻结设计  
> 适用角色：超级管理员、管理员、游客

## 1. 页面范围

系统管理第一版包含四个页面：用户管理、系统参数、操作日志、备份与恢复。安全审计日志不提供删除和编辑入口，仅超级管理员可按高风险事件查询。

## 2. 权限矩阵

| 功能 | 超级管理员 | 管理员 | 游客 |
|---|---:|---:|---:|
| 用户列表与详情 | 查看/新增/编辑/停用 | 无权限 | 无权限 |
| 角色调整与密码重置 | 允许 | 无权限 | 无权限 |
| 系统参数 | 查看/修改 | 只读必要公开参数 | 只读必要公开参数 |
| 操作日志 | 查看/导出/隐藏/恢复 | 查看本人操作 | 无权限 |
| 安全审计日志 | 查看/导出 | 无权限 | 无权限 |
| 手动备份 | 允许 | 无权限 | 无权限 |
| 恢复备份 | 允许，二次确认 | 无权限 | 无权限 |
| 提成 | 查看/登记/修改/删除/导出 | 完全不可见 | 完全不可见 |

权限必须由后端守卫和查询字段白名单执行，不能只靠前端隐藏菜单或按钮。

## 3. 用户管理页

### 3.1 列表

筛选项：登录名、显示姓名、角色、账号状态。列表字段：登录名、姓名、角色、状态、最后登录时间、创建时间和操作。

新增或编辑用户包含：登录名、显示姓名、角色、联系电话、账号状态和初始密码。承租人、业主等业务对象不是系统用户，不在此页创建。

### 3.2 关键规则

- 登录名唯一，不允许通过软删除后重复制造身份混淆。
- 不允许停用、删除或降级最后一名启用状态的超级管理员。
- 用户停用或重置密码后，立即撤销其全部刷新令牌。
- 超级管理员不能删除当前正在使用的自己；如需交接，先新增或启用另一名超级管理员。
- 修改角色、停用账号、重置密码均写入安全审计日志。
- 密码字段、令牌哈希永不进入普通操作日志、导出或接口响应。

## 4. 系统参数页

第一版允许超级管理员配置：项目名称、提前催租天数（默认 7）、合同到期提醒天数（默认 30）、长期空置天数（默认 30）、收据编号前缀、上传文件大小限制、默认承租人类型（个人）和默认缴租周期。

以下内容不是可编辑参数：角色权限、租金计算公式、收租率口径、押金和预收款财务归类、审批流程、审计保留规则。修改参数时展示字段说明，保存后立即记录修改前后值。

## 5. 操作日志页

### 5.1 查询

支持按日期范围、模块、操作类型、操作人、角色、业务对象编号和关键词筛选。列表展示时间、操作人、模块、动作、对象、摘要、原因和来源 IP；详情抽屉展示脱敏后的修改前后值。

### 5.2 删除与恢复

- 只有超级管理员能执行“删除日志”。
- 删除实际把普通操作日志标记为隐藏，不物理删除。
- 删除前必须填写原因并二次确认；隐藏后的日志默认不在普通列表显示。
- 超级管理员可切换“包含已删除”并恢复日志。
- 删除和恢复动作都写入不可删除的安全审计日志。

## 6. 安全审计页

记录已确认收款修改、收款作废与退款、优惠审批、退差确认、押金退还、提成变更、日志删除恢复、敏感导出、用户权限变化和数据库恢复等高风险事件。

- 仅追加，不提供编辑和删除接口。
- 详情展示事件类型、对象、操作人、原因、时间及修改前后快照。
- 采用前一条哈希和当前记录哈希形成校验链，定期执行完整性检查。
- 普通管理员和游客不能访问该页面或接口。

## 7. 备份与恢复页

列表展示备份编号、类型、状态、数据大小、校验值、创建时间、保留期限和恢复状态。系统每天自动备份数据库及附件清单，保留最近 30 天；超级管理员可以手动创建备份。

恢复流程：选择成功备份 → 查看时间和校验信息 → 输入恢复原因 → 二次确认 → 自动备份当前数据 → 执行恢复 → 校验数据库与附件清单 → 强制所有用户重新登录 → 写入安全审计日志。

恢复失败时不得覆盖原备份记录，必须保留失败原因和恢复前备份，以便再次处理。

## 8. 接口边界

- `/admin/users/**`、`/admin/settings/**`、`/admin/backups/**`：仅超级管理员。
- `/admin/operation-logs`：超级管理员全部；管理员仅允许查询自己的日志且字段受限。
- `/admin/security-audits/**`：仅超级管理员，只读。
- 所有接口统一分页、参数校验和权限拒绝响应；禁止通过修改 URL 或请求参数越权。
- 下载日志或审计导出文件前再次校验当前用户权限，临时下载地址短期有效。

## 9. 验收标准

- 最后一名启用状态的超级管理员无法被停用或降级。
- 管理员和游客不能获得提成、财务中心或安全审计数据。
- 管理员只能查询本人操作日志，无法查看其他用户的敏感操作快照。
- 删除普通操作日志后可恢复，且删除和恢复均存在不可删除审计记录。
- 高风险记录不可通过业务接口修改或删除，哈希链校验能发现直接篡改。
- 备份恢复前自动创建当前数据备份，恢复后全部旧登录状态失效。
- 系统参数修改前后值和操作人可追溯。
