# Task008 优惠减免、退款与作废验收记录

## 已实现

- 账单调整申请：一次性优惠、减免、补收、更正；管理员提交，超级管理员确认或驳回。
- 待审批或被驳回调整不会影响账单；确认后在事务中重算账单应收、未收、状态及合同缴费快照。
- 收款退款：退款申请逐条关联原收款分配和回退金额；确认后保留原收款、原分配和退款记录，只累计回退金额并重算账单。
- 整笔收款作废：管理员提交申请，超级管理员确认；确认后逆转该收款的账单分配及预收款入账，原收据编号保留并标记作废。
- 收款页提供优惠/减免、退款、作废申请及超级管理员审批入口。

## 权限

- 管理员和超级管理员可提交调整、退款、作废申请。
- 仅超级管理员可确认或驳回调整、退款、作废申请。
- 全部接口由 JWT 与角色守卫在后端强制校验。

## 数据结构

- `bill_adjustments`、`payment_refunds`、`payment_void_requests`。
- 经确认新增 `payment_refund_allocations`，保存部分退款逐条回退原收款分配的金额。

## 已知边界

- 预收款退款不混入普通收款退款，留给后续预收款/退租结算处理，避免未知资金来源被错误扣减。
- 与单笔收款绑定的一次性优惠的作废联动将在后续完善其“绑定收款”业务入口后实现；当前合同预设优惠不与单笔收款绑定。
- 押金退款、退租结算和阶梯退差不属于本任务。
