# Task013 系统设置、日志与备份验收记录

## 已实现

- 超级管理员可查看和修改冻结范围内的系统参数：项目名称、提醒天数、长期空置阈值、收据前缀、上传上限、默认租客类型和默认付款周期。
- 金额口径、角色权限、审批流程、财务分类和审计保留规则不会出现在可编辑参数中。
- 系统参数修改会记录操作日志（变更前后值）和带 SHA-256 链式哈希的安全审计记录。
- 操作日志提供超级管理员查看、隐藏（非删除）和恢复；隐藏、恢复必须填写原因且会进入安全审计。
- 管理员后端只能读取自身的受限操作日志；访客无法访问。系统参数、备份和安全审计均由后端角色守卫限制为超级管理员。
- 提供安全审计只读列表和完整性校验接口，可发现带哈希记录遭篡改或链路断裂。
- 新增备份记录模型、30 天保留期字段、手工备份与每日自动备份。数据库 SQL 与 `uploads` 附件目录 ZIP 会一并归档到受控备份目录，记录校验值；到期清理只处理系统生成的 SQL、ZIP 与清单。
- 恢复仅接受校验通过的成功备份，必须填写原因和输入“确认恢复”；系统先创建恢复前备份，恢复成功后撤销全部刷新令牌，并记录不可删除安全审计。
- 前端新增 `/admin/system` 中文系统管理页，包含系统参数、操作日志、安全审计、备份与恢复四个标签页。
