# Task005 承租人与文件基础验收记录

## 已实现

- 个人和单位承租人新增、编辑、分页查询与状态管理。
- 证件号码使用环境变量密钥 AES-256-GCM 加密保存，以 HMAC-SHA256 精确查重；普通接口只返回末四位脱敏值。
- 完整证件号仅允许超级管理员和管理员在专用接口查看，并写入安全审计事件；游客没有该入口。
- 证件附件使用不可预测文件名保存到运行期 `uploads/tenant-ids`，数据库只存元数据；上传验证配置的 MIME 类型、文件签名和大小，下载经过鉴权接口。
- 本机 `.env` 已配置证件加密、摘要密钥，以及 10MB / JPG、PNG、HEIC、PDF 的附件限制；密钥不记录在本文档或源码中。

## 验证

- MySQL Task005 migration 已应用，`prisma migrate status` 显示数据库为最新状态。
- `npm run build`、`npm run lint`、`npm run db:validate` 通过。
- `npm --prefix backend run test`：5 个测试套件、12 个测试通过，包含加密/解密与格式归一化 HMAC 测试。

## 手工验收

1. 登录后进入“承租人管理”，新增个人或单位承租人。
2. 确认证件号码在列表只显示末四位；管理员点击“查看证件”才可获得完整值。
3. 上传一份符合限制的证件附件，再通过下载接口获取；伪造扩展名或超过 10MB 的文件应被拒绝。
4. 用游客账号验证其只能查看脱敏资料，直接访问新增、编辑、完整证件和上传接口也会被后端拒绝。
