# SRMS 手工验收清单

本清单用于完成尚未以真实业务数据验收的 Task006 至 Task013。请优先在测试库或用明确标记为“测试”的房源、承租人和合同执行；不要在未备份的正式数据上测试退款、退租或数据库恢复。

## 开始前

1. 登录超级管理员账号，确认首页可以打开。
2. 打开“房源管理”，准备一间空置测试房源；打开“承租人管理”，准备一名测试承租人。
3. 记录测试合同号、账单号和收据号，便于后续逐项核对。
4. 每完成一个步骤，在页面和对应的操作日志中核对操作人、时间、对象编号与金额。

## 合同、账单与优惠（Task006、Task008）

1. 在“合同管理”创建固定月租合同，租期最后一天选择非整月日期。
2. 核对最后一期租金是否严格按“月租 ÷ 30 × 实际计租天数”计算。
3. 为合同新增日期区间免租、固定金额优惠和比例优惠，核对每期账单的原始租金、减免金额和净应收。
4. 尝试在已经收款的账期之前确认合同变更，系统应拒绝或锁定已收款账单；未收款的后续账单才可重算。
5. 用管理员提交变更，再用超级管理员确认；分别核对变更记录、账单快照和操作日志。

## 收款、退款与预收款（Task007、Task008）

1. 在“收款管理”选择测试合同账单，录入一笔等于应收的收款并确认。
2. 核对账单有效实收、未收金额和收款状态；再录入超额金额，核对超额部分进入预收款而非租金实收。
3. 为已确认收款提交退款申请，逐条填写原账单分配的回退金额。
4. 用超级管理员确认退款，核对原收款和原分配仍保留、退款记录新增、账单实收被正确回退。
5. 尝试退款超过原分配金额，系统必须拒绝。

## 阶梯退差与退租押金（Task009、Task010）

1. 创建包含至少两个阶梯的测试合同，并完成达到阶梯条件所需的收款。
2. 在“阶梯退差”提交申请；超级管理员确认后，核对差额金额、收款/预收款去向和审计记录。
3. 在“退租结算”创建结算，分别填写欠租、维修或其他扣款依据。
4. 提交并确认结算后，核对应退押金、待补交金额、未来账单作废状态和房源状态。
5. 确认押金退款后，核对合同结束、房源释放，以及所有金额没有被物理删除。

## 财务、导出与驾驶舱（Task011、Task012）

1. 打开“财务中心”，按同一日期范围核对租金收缴、资金流水与页面中的收款/退款记录一致。
2. 创建 Excel 和 PDF 导出任务，等待任务变为“成功”后下载文件；核对筛选日期、操作人和导出内容。
3. 用管理员或游客账号访问财务/提成接口和页面，确认提成数据不会显示或下载。
4. 打开首页驾驶舱，变更楼栋、房态和月份筛选，核对汇总数量与房源、账单数据一致。

## 系统设置、日志与备份（Task013）

1. 在“系统管理”修改一个非冻结系统参数，核对操作日志的修改前后值，以及安全审计链完整性。
2. 分别用超级管理员、管理员和游客测试系统管理入口；后两者不应获得系统设置、备份和安全审计权限。
3. 在确认 `BACKUP_DIR` 位于受控磁盘后，创建一次手动备份，核对数据库 SQL、附件 ZIP、清单和校验值。
4. 恢复操作只建议在测试库执行：先确认自动生成恢复前备份，再输入原因和“确认恢复”，最后重新登录并核对审计记录。

## 验收结论

- 任一步骤与冻结需求不一致时，请记录页面路径、操作时间、测试数据编号、截图和实际结果；不要通过修改金额或直接改数据库绕过问题。
- 全部步骤通过后，可将 README 中标记为“待真实数据手工验收”的 Task006 至 Task013 更新为完成。
