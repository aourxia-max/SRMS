# Task010 押金与退租验收记录

## 已实现

- 押金收取以独立 `DEPOSIT` 收款和 `deposit_transactions` 正向余额流水保存，不进入租金账单分配或租金实收。
- 发起退租后合同和房源进入 `PENDING_CHECKOUT`；未来账单不在发起时删除或作废。
- 管理员提交结算时录入实际退房、验房、欠租及验房扣款。欠租必须关联有效账单；维修、损坏、清洁和其他扣款必须填写验收记录引用。
- 超级管理员确认结算后，按欠租优先、验房扣款其次抵扣押金，写入不可覆盖的押金流水，并作废实际退房日之后未收款的未来账单。
- 无应退押金、无待补交且预收款余额为零时，结算确认后直接结束合同并将房态更新为结算单锁定的目标状态。
- 管理员登记押金退款，必须上传 `DEPOSIT_REFUND_PROOF` 凭证；超级管理员确认后锁定凭证、写负向押金流水、完成结算、结束合同并更新房态。
- 前端入口：`/checkout`，提供押金收取、发起退租、结算项目、结算确认、凭证上传和退款确认入口。
