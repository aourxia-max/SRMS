# SRMS 合同、收款与退差关键页面设计（V1.1）

## 1. 页面范围

本设计覆盖五个关键页面：

1. 新增合同。
2. 阶梯合同详情。
3. 阶梯退差确认。
4. 固定月租合同详情。
5. 固定月租合同退差确认。

所有页面共同遵循合同固定最长租期、合同级计价快照、实际退差自由录入、管理员登记与超级管理员确认财务动作等业务规则。

## 2. 新增合同

### 2.1 页面目标

管理员在一个页面完成房源、承租人、租期、押金、计价方式、免租优惠和合同附件录入，确认后生成合同和全租期账单。

### 2.2 页面分区

| 分区 | 主要内容 |
|---|---|
| 基本信息 | 房源、主/副承租人、开始/结束日期、入住日期、押金、缴租周期 |
| 计价方式 | 固定月租或自定义弹性阶梯 |
| 自定义档位 | 档位名称、达标月数、月租、是否需结清、排序、增删 |
| 其他约定 | 提前退租、免租优惠、合同附件、备注 |
| 租房提成 | 仅超级管理员可见；填写所属对象和金额 |
| 合同摘要 | 房源、承租人、最长租期、初始月租、押金 |
| 账单预览 | 每个账期采用的档位和月租快照 |

### 2.3 核心交互

- 选择固定月租时隐藏阶梯表格，只填写固定月租。
- 选择弹性阶梯时可以不使用模板，直接新增任意档位。
- 使用模板后只带出数据，允许修改当前合同档位，不回写模板。
- 点击“新增档位”增加一行；删除后自动重新排序。
- 达标月数必须为正整数且不能重复。
- 后一档月租高于前一档时提示但允许继续。
- 合同结束日期决定最长可生成账期数。
- 租缴周期可选择1至12个月，表示合同约定每隔几个月应缴一次。
- 租缴周期不限制单次实际支付覆盖的账期数；例如周期为1个月，仍允许某次一次支付5个月。
- 点击确认时先展示账单预览，再生成合同和账单。
- 租房提成不参与租金、押金、优惠或账单计算，超级管理员可在合同生效前后随时修改。
- 管理员和游客的页面及接口响应中完全不返回提成字段。

### 2.4 保存规则

- 保存草稿不占用房源，不生成正式账单。
- 确认合同前锁定房源并校验租期冲突。
- 确认后复制合同价格快照，生成全租期账单。
- 房源状态从空置变为待入住。
- 已出售、待出售、停用和存在冲突合同的房源不能选择。

## 3. 合同详情

### 3.1 页面目标

集中展示合同履行、价格档位、账单、收款、退差、成员、附件和操作日志，并提供收款、变更和退租入口。

### 3.2 顶部信息

- 合同编号和房号。
- 主承租人。
- 合同开始、结束日期。
- 合同状态。
- 当前档位和月租。
- 已连续完整账期。
- 下一档预计达标日期。

### 3.3 价格阶梯

使用时间线和表格共同展示：

- 已完成档位。
- 当前档位。
- 未来档位。
- 每档达标月数、合同月租、预计/实际达标日期。
- 系统参考退差和已经确认的实际退差。

档位数据来自合同价格快照，不直接读取可能已经修改的价格模板。

### 3.4 退差记录

每条记录显示：

- 达标档位。
- 系统参考退差。
- 实际退差。
- 差异原因。
- 实际退款或转预收款。
- 处理状态。
- 原退差与补充退差关联。

已确认记录不能编辑。需要追加金额时使用“登记补充退差”。

### 3.5 快捷操作

| 操作 | 权限与规则 |
|---|---|
| 登记收款 | 超级管理员、管理员 |
| 合同变更 | 管理员登记，超级管理员确认 |
| 发起退租 | 超级管理员、管理员 |
| 登记补充退差 | 管理员登记，超级管理员确认 |
| 查看财务汇总 | 仅超级管理员 |
| 打印合同 | 按角色和敏感信息权限控制 |

## 4. 阶梯退差确认

### 4.1 页面目标

让超级管理员在一个页面完成资格、金额、差异原因、处理方式和凭证核验，避免只看到管理员填写的最终金额。

### 4.2 达标核验

页面显示系统校验结果：

- 连续完整账期是否达标。
- 阶段账单是否按合同要求结清。
- 合同是否处于履行中。
- 同档位是否已经存在有效退差。
- 实际退差是否超过合同累计有效实收。

任何硬性校验失败时，不显示确认按钮。

### 4.3 金额核对

并列展示：

| 金额 | 说明 |
|---|---|
| 系统参考退差 | 可以为空，只用于核对 |
| 管理员录入实际退差 | 本次真正执行的金额 |
| 与参考值差额 | 实际金额减参考金额 |

实际值与参考值不一致时，差异原因必填并突出显示。下方列出参与计算的历史账单和有效实收。

### 4.4 处理方式

#### 实际退款

- 填写退款日期、退款方式、备注。
- 必须上传转账截图；现金上传租客签字凭证。
- 超级管理员可以查看原图。
- 确认后锁定凭证并生成退款流水。

#### 转为预收款

- 不要求上传退款截图。
- 确认后生成预收款入账流水。
- 后续按最早到期账单优先抵扣。

### 4.5 审批结果

- 确认：退差生效，写财务流水和安全审计日志。
- 驳回：必须填写原因，退回管理员修改后重新提交。
- 撤销已确认退差：仅超级管理员，生成逆向流水，不删除原记录。

## 5. 悬停说明

以下字段显示“?”并支持鼠标悬停查看完整说明：

- 合同结束日期。
- 计价方式。
- 达标月数。
- 账单价格预览。
- 系统参考退差。
- 实际退差与差额。
- 退款凭证锁定规则。

移动端没有鼠标时，点击“?”打开说明气泡。

## 6. 通用验收标准

- 新增合同能够自由增加、删除和修改价格档位。
- 固定月租和弹性阶梯切换正确。
- 账单预览与最终价格快照一致。
- 合同详情能准确展示当前档位和下一档日期。
- 实际退差与参考值不同时强制填写原因。
- 超级管理员能够看到退差涉及的全部账单和实收。
- 实际退款没有凭证时不能确认。
- 转预收款时不要求凭证，并正确生成余额流水。
- 补充退差不会覆盖原退差。
- 五个页面的重要操作全部记录操作日志和安全审计。

## 7. 固定月租合同详情

### 7.1 页面目标

集中展示固定月租合同、逐月账单、收款记录和一笔收款的账单分配关系，让管理员清楚区分“合同约定租缴周期”和“本次实际覆盖账期”。

### 7.2 一次缴纳多个月

以合同约定每1个月缴租、固定月租1200元为例：

1. 第一个月收款1200元，分配至第1个月账单。
2. 第二个月租客一次支付6000元，系统只创建一笔收款。
3. 该收款通过五条分配明细，分别结清第2至第6个月的五张账单。
4. `paid_through_date`更新至第6个月账期末，`next_due_date`更新为第7个月账期开始日。
5. 合同的`payment_cycle_months`仍为1，不因本次提前多缴而改变。

默认先冲抵最早到期欠款。用户主动选择未来账单时，页面展示账期及每期应收；若未来账单金额不同，按所选账单实际金额合计，不用“当前月租×月数”估算。超过选中账单应收的金额进入预收款。

## 8. 固定月租合同退差确认

### 8.1 适用范围

固定月租合同没有阶梯达标条件和系统目标价。发生房屋维修、履约瑕疵或其他双方协商减租时，可由管理员登记“固定合同手工退差”，超级管理员确认。

页面展示：

- 合同状态、涉及账单及有效实收。
- 管理员录入的实际退差金额和必填原因。
- 实际退款或转预收款方式。
- 实际退款日期、方式、备注和退款凭证。
- 提交人、提交时间及超级管理员意见。

固定月租退差的系统参考金额显示“无”，不虚构自动计算值。退差生成独立流水，不修改原账单和原收款；若只是收款录错，必须使用收款作废或收款退款流程。

## 9. 本次新增验收标准

- 新增合同的租缴周期可选择1至12个月。
- 租缴周期为1个月时，允许一笔收款覆盖任意多张未结账单。
- 一笔收款与多张账单的分配金额可追溯且合计一致。
- 提前多缴后合同约定租缴周期保持不变。
- 固定月租退差不显示虚假的系统参考金额。
- 固定月租退差必须关联账单、填写原因并由超级管理员确认。
- 实际退款没有凭证时不能确认；转预收款正确生成余额流水。
- 所有财务操作写入操作日志和不可篡改的安全审计日志。
