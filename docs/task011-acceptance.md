# Task011 财务中心与提成验收记录

## 已实现

- 财务中心后端接口仅允许超级管理员访问，管理员和游客由后端角色守卫拒绝。
- 租金收缴按账期展示原应收、优惠减免、净应收、有效实收、未收与收租率；押金和预收款不计入租金收入。
- 资金流水按实际发生日展示租金收款、押金收取、收款退款和押金内部抵扣；内部抵扣明确标记为非外部现金流。
- 提成台账仅包含合同、所属对象和金额；可在任意合同状态下新增、修改、删除，不产生付款状态或资金流水。
- 每次提成新增、修改、删除均写入不可删除的 `security_audit_logs`，保存修改前后数据。
- 前端入口：`/finance`，包含收缴报表、资金流水和提成台账。

## 数据库

- 已应用 `20260723160000_task011_finance_commissions`；当前 MySQL 共 14 个 migration。

## 验证

- `npm run build`、`npm run lint`、`npm run db:validate` 通过。
- `npm run test` 通过：10 个测试套件、30 个测试。
- `npm --prefix backend run test:e2e -- --runInBand` 通过：1 个测试套件、6 个接口测试。
- 后端健康检查返回 200；未登录访问 `/api/finance/rent-collection` 返回 401。

## 已知边界

- 财务导出采用异步任务：任务创建后生成 Excel/PDF，完成后在财务中心任务列表下载；创建、查询和下载接口均要求超级管理员身份。报表仍复用财务查询服务的筛选口径。
- 当前资金流水已覆盖本阶段已落地的核心流水类型；预收款退款登记入口尚未完成，因此不虚构该类退款流水。
