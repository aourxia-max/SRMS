# SRMS 退租结算与押金退还页面设计（V1.0）

## 1. 页面范围

本设计覆盖发起退租、退租结算、押金退还确认三个页面。

## 2. 发起退租

管理员选择合同，填写退租类型、计划退房日期、房屋交接日期、退租原因、验收时间和退房后目标房源状态。

发起后：

- 合同变为`PENDING_CHECKOUT`。
- 房源变为`PENDING_CHECKOUT`。
- 未来账单暂不删除，待结算确认后按实际退房日作废。
- 系统展示押金、欠租、预收款和未来账单快照。

## 3. 退租结算

房屋验收完成后，管理员登记欠租、维修扣款和其他双方确认的扣款项目。第一版不包含水电费。

计算口径：

```text
押金可退金额 = 押金余额 - 抵扣欠租 - 已确认的其他扣款
租客仍需补交 = max(0, 欠租与扣款合计 - 押金余额)
最终退款合计 = 押金可退金额 + 应退预收款
```

扣款必须关联欠租账单、房屋验收记录或上传的证明材料。结算确认后锁定金额、作废退房日之后的未来账单，但合同仍保持待退房。

如租客仍需补交，必须完成补收后才能进入退款或合同结束。预收款余额也必须完成退款或其他经批准的处理。

## 4. 押金退还确认

管理员登记、超级管理员确认。页面不要求填写收款人、退款账户后四位或交易流水号，只填写退款日期、方式和备注。

实际退还押金必须上传交易截图；现金退款上传租客签字凭证。确认时校验：

- 房屋已完成交接。
- 结算单已经确认。
- 欠租和扣款已结清或用押金抵扣。
- 租客待补交金额为0。
- 预收款余额已处理。
- 退款金额等于结算单锁定金额。
- 至少存在一张有效退款凭证。

超级管理员点击“确认退还并结束合同”后，在同一事务中写押金退款流水、锁定凭证、清零押金余额、完成结算、结束合同并将房源更新为验收时选定的状态。

如没有可退押金且押金全部抵扣，结算确认通过上述结束条件后可直接结束合同，无需伪造0元退款记录或上传退款截图。

## 5. 撤销与审计

已确认退款不能删除或直接修改。确需纠错时，由超级管理员发起逆向流水并填写原因，原退款、凭证和结算记录永久保留。普通操作日志可以由超级管理员隐藏，但对应安全审计日志不可删除。

## 6. 验收标准

- 发起退租不会提前作废未来账单或结束合同。
- 押金可正确抵扣欠租和经确认的验收扣款。
- 扣款没有依据时不能提交结算。
- 结算确认后才作废实际退房日之后的未来账单。
- 存在待补交、未处理预收款或未退款押金时不能结束合同。
- 押金退款没有凭证时不能确认。
- 不要求收款人、账户后四位和交易流水号。
- 退款确认后合同结束、押金余额归零、房源状态正确更新。
