# SRMS 自定义弹性租期阶梯计价规则（V1.1）

## 1. 功能定义

该功能适用于租客无法预先确定实际租住时长的场景。合同仍设置固定的最长开始、结束日期，并可约定提前退租；每份合同可以自由录入租期档位、档位月租和实际退差金额。

系统不固定1个月、3个月、6个月、12个月，也不固定800元、900元、1000元、1100元。上述数据只能作为某一次合同的示例。

本功能只对选择“自定义弹性阶梯计价”的合同生效，普通固定月租合同不受影响。

## 2. 合同级自定义档位

管理员可以在合同中新增任意数量的档位：

| 字段 | 是否必填 | 说明 |
|---|---:|---|
| 档位名称 | 是 | 例如短月租、季租、半年租，也可以自定义 |
| 达标月数 | 是 | 正整数，可录入1、2、3、5、6、9、12等 |
| 档位月租 | 是 | 自由输入，保留两位小数 |
| 是否要求阶段账单结清 | 是 | 默认“是” |
| 备注 | 否 | 记录协商约定 |

规则：

- 达标月数不能重复，必须按从小到大排列。
- 月租金额不能小于0。
- 后一档价格高于前一档时系统给出提示，但允许继续保存。
- 可以选择常用价格模板快速带出，再针对本合同修改。
- 合同确认后保存独立价格快照；以后修改模板不影响该合同。

例如某合同可以录入：

| 自定义档位 | 达标月数 | 月租 |
|---|---:|---:|
| 短租档 | 1 | 1100元 |
| 季租档 | 3 | 1000元 |
| 半年档 | 6 | 900元 |
| 年租档 | 12 | 800元 |

另一份合同也可以录入2个月1200元、5个月1050元、9个月850元，系统不限制为固定组合。

## 3. 账单生成

合同确认时一次生成合同期内全部账单，每张账单保存其使用的合同档位和月租快照。

如果合同录入1/3/6/12个月四档，系统按该合同的档位边界生成各期账单；如果录入2/5/9个月，则按2/5/9个月边界生成。

租客提前退租时，系统作废实际退租日之后尚未发生的未来账单，保留所有历史账单和收款记录。

## 4. 达标条件

默认同时满足以下条件才进入新档位：

1. 从合同开始日连续租住满该档位要求的完整账期。
2. 档位设置要求结清时，对应阶段账单全部结清。
3. 合同状态为履行中。
4. 合同没有进入待退房、作废或结束状态。
5. 同一档位以前没有完成过相同退差。

不足整月不计入达标月数。部分付款不算结清；补齐后系统重新检查。

## 5. 退差金额

退差金额根据管理员与租客的实际协商结果录入，不由系统固定公式决定。

退差单同时保存以下数据：

| 字段 | 说明 |
|---|---|
| 系统参考退差 | 系统根据合同账单和档位价格计算，仅供参考，可为空 |
| 实际退差金额 | 管理员根据实际情况输入的金额 |
| 差额 | 实际退差减去系统参考退差 |
| 差异原因 | 实际值与参考值不一致时必须填写 |
| 处理方式 | 实际退款或转为预收款 |
| 备注 | 记录与租客协商内容 |

系统可以提供参考公式：

```text
参考退差
= 达标期原始账单累计金额
  - 档位月租 × 达标完整月数
  - 此前已确认的阶梯退差
```

管理员可以采用参考金额，也可以根据实际约定录入其他金额。超级管理员确认页面必须同时显示原账单、参考金额、实际金额、差额和差异原因。

### 示例

前三个月各收1100元，租客满3个月后，双方可以协商实际退差300元，也可以根据其他约定填写200元、350元或不退。系统不自动强制为300元。

实际退差不得小于0。原则上不得超过该合同累计有效实收租金；如果需要额外补贴，应作为独立费用处理，不混入租金退差。

## 6. 退差处理方式

### 6.1 实际退款

1. 系统生成待处理退差单或管理员手工发起。
2. 管理员填写实际退差金额、退款日期、退款方式、差异原因和备注。
3. 上传转账截图；现金退还上传承租人签字凭证照片。
4. 提交超级管理员确认。
5. 确认后退差生效，凭证锁定。
6. 错误时撤销原退差单后重新登记，不能修改已确认记录。

### 6.2 转为预收款

1. 管理员填写实际退差金额并选择“转预收款”。
2. 提交超级管理员确认。
3. 确认后写入合同预收款流水。
4. 系统按最早到期账单优先抵扣。

转预收款不要求退款截图，但必须保存登记人、确认人、金额、差异原因、备注和操作时间。

### 6.3 补充退差

同一档位确认退差后，如果双方再次协商增加退款，不能覆盖原记录。管理员创建“补充退差单”，关联原退差单，填写补充金额和原因，再由超级管理员确认。

## 7. 权限

| 操作 | 超级管理员 | 管理员 | 游客 |
|---|---:|---:|---:|
| 创建常用价格模板 | 是 | 否 | 否 |
| 在合同中自定义档位和价格 | 是 | 是 | 否 |
| 查看系统参考退差 | 是 | 是 | 否 |
| 输入实际退差金额 | 是 | 是 | 否 |
| 上传退款凭证 | 是 | 是 | 否 |
| 确认、驳回或撤销退差 | 是 | 否 | 否 |
| 查看财务汇总 | 是 | 否 | 否 |

模板修改、合同价格修改、退差登记、确认、驳回、撤销、补充退差和凭证上传都写入操作日志。确认和撤销同时写入不可删除的安全审计日志。

## 8. 页面设计

### 8.1 合同表单

计价方式包括：

- 固定月租。
- 自定义弹性阶梯计价。

选择弹性阶梯后，显示可增删的档位表格。管理员可以逐行输入档位名称、达标月数、月租和结清条件，也可以从模板带出后修改。

### 8.2 合同详情

显示：

- 当前档位和当前月租。
- 全部自定义档位。
- 已连续租住完整月数。
- 下一档位和预计达标日期。
- 系统参考退差，仅作提示。
- 历次实际退差、补充退差、处理方式和凭证。

### 8.3 退差确认

超级管理员确认页面并列显示：

- 达标档位和账期。
- 原账单及有效收款。
- 系统参考金额。
- 管理员输入的实际金额。
- 两者差额和差异原因。
- 实际退款凭证或转预收款说明。

## 9. 驾驶舱待办

- 7天内即将达到自定义档位。
- 已达标但阶段账单尚未结清。
- 已达标待登记实际退差。
- 阶梯退差待超级管理员确认。
- 实际退款凭证待审核。
- 补充退差待确认。

## 10. 特殊场景

| 场景 | 处理规则 |
|---|---|
| 合同没有设置阶梯 | 按固定月租处理 |
| 档位为2、5、9个月 | 按合同自定义月份判断，不套用3、6、12个月 |
| 达标但账单未结清 | 默认暂不转档；合同档位可单独配置是否必须结清 |
| 系统参考300元，双方约定退200元 | 录入200元并填写差异原因，超级管理员确认 |
| 双方约定不退款 | 实际退差录入0元并填写原因，可只更新达标档位 |
| 已退200元后又决定补退100元 | 新建补充退差单，不修改原200元记录 |
| 一次预付多个月 | 按各账单价格快照分配，多余金额进入预收款 |
| 合同优惠与阶梯价格同时存在 | 先采用合同档位价格，再计算免租和优惠；参考退差展示计算明细 |
| 价格模板后续调整 | 不影响已有合同价格快照 |
| 系统任务重复执行 | 合同、档位和退差类型唯一约束避免重复生成 |

## 11. 验收标准

- 可以自由新增、删除、排序合同价格档位。
- 档位月数和月租均可输入，不限制为固定组合。
- 不选择模板也能创建弹性阶梯合同。
- 选择模板后可以修改本合同档位，且不影响模板和其他合同。
- 系统参考退差可显示，也可以设置为不计算。
- 管理员能够自由输入实际退差金额。
- 实际值与参考值不一致时必须填写原因。
- 实际退款未上传凭证不能提交确认。
- 转预收款后余额和后续账单抵扣正确。
- 补充退差不覆盖原退差记录。
- 撤销退差后，退款、预收款、档位和日志全部正确回退。
- 并发执行达标任务时不会生成重复退差。
